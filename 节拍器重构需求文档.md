# Flutter 节拍器下一轮优化需求文档

请基于当前 Flutter 节拍器项目继续优化。本轮重点是 WebView 页面浏览体验、节拍器主页配置入口重组、设置页重新定位，以及调音器 UI 稳定性。

## 1. WebView 页面返回能力

当前左侧菜单页面是内嵌 WebView，加载用户配置的网页。现在的问题是进入网页深层页面后，无法返回网页上一层。

优化要求：

- 在 WebView 页面左上角或底部添加一个返回按钮。
- 点击返回按钮时：
  - 如果 WebView 可以后退，则执行网页后退。
  - 如果 WebView 不能后退，则什么都不做，或给出轻量提示。
- 可选增强：
  - 拦截 Android 系统返回键。
  - 如果当前在 WebView 页面且 WebView 可以后退，则优先网页后退。
  - 如果 WebView 不能后退，再按 App 原有返回逻辑处理。

## 2. WebView 页面悬浮节拍器按钮优化

当前 WebView 页面右下角有节拍器 Start / Stop 悬浮按钮，但可能遮挡网页内容。

优化要求：

- 将悬浮节拍器按钮改成半透明样式。
- 按钮仍然需要明显可点击，但不要过度遮挡网页内容。
- 播放状态仍然要和节拍器主页 Start / Stop 保持同步。
- 可选增强：
  - 按钮闲置时降低透明度。
  - 用户点击或拖动时提高透明度。
  - 支持拖动到其他位置，避免遮挡网页关键内容。

## 3. 底部左侧菜单改为可自定义网页

当前底部左侧菜单叫“吉他社”，并固定打开 `https://www.jitashe.org/`。

新的产品方向：

- 左侧菜单不应该固定叫“吉他社”。
- 左侧菜单应表示“自定义网页 / 网页 / 社区 / 工具页”等更通用含义。
- 用户可以在设置页面自定义该网页地址。

优化要求：

- 底部左侧菜单名称改为更通用的名称，例如：
  - `网页`
  - `社区`
  - `工具`
  - 最终命名可根据现有 UI 风格决定。
- 默认网页地址仍可使用：
  - `https://www.jitashe.org/`
- 设置页面提供网页地址配置入口。
- 用户修改地址后：
  - WebView 页面加载新地址。
  - 地址需要本地持久化。
  - 下次打开 App 时使用用户保存的地址。
- 需要处理用户输入：
  - 如果未输入协议，可以自动补全 `https://`。
  - 如果地址为空，保留默认地址或提示用户。

## 4. 设置页面重新定位

当前设置页面只有“保存节拍器配置”和“保存的配置列表”，使用起来有割裂感。

新的结构方向：

- “保存配置”和“加载配置”更适合放在节拍器主页。
- 设置页面应该放 App 级设置、偏好项、网页地址等内容。

优化要求：

- 将“保存当前节拍器配置”和“加载已保存配置”的入口，从设置页面移到节拍器主页。
- 建议放在 BPM 轮盘的左右两侧：
  - 左侧：保存当前配置。
  - 右侧：加载/选择已保存配置。
- 保存配置仍需支持输入配置名称。
- 加载配置仍需展示已保存配置列表，并支持恢复配置。
- 配置数据仍然使用当前本地持久化方案。

设置页面可放入：

- 震动反馈开关。
- 自定义左侧网页菜单的链接地址。
- 未来其他 App 级偏好设置。

设置页面后续规划：

- 练习统计以后放到独立页面或设置相关页面。
- 后续可实现类似 GitHub contribution calendar 的练琴日历：
  - 每一天一个格子。
  - 当天有练琴记录时上色。
  - 练琴时长越长颜色越深。
- 本轮可以先预留入口，不一定要完整实现练琴日历。

## 5. 节拍器主页配置入口优化

为了让保存/加载配置与节拍器操作更连贯，需要在主页添加配置相关快捷入口。

优化要求：

- 在 BPM 轮盘左右两边添加两个功能入口：
  - 保存当前配置。
  - 加载已保存配置。
- 不要让主页显得拥挤。
- 按钮可以是图标按钮或小型半透明操作按钮。
- 需要适配小屏幕：
  - 如果左右空间不足，可以放到轮盘下方左右两侧。
  - 不要遮挡 BPM 数字、Tap Tempo、`+/-` 微调按钮。
- 保存和加载操作应使用抽屉或弹窗：
  - 保存：输入配置名称并确认。
  - 加载：展示配置列表，点击恢复。

## 6. 调音器抽屉 UI 稳定性优化

当前调音器问题：

- 没有声音时显示 `Listening`。
- 有声音时显示音名/频率并跳动。
- 在有声/无声之间切换时 UI 反复跳变，观感不稳定。

优化方向：

- 调音器 UI 应像常见调音器一样常驻显示主界面。
- 不要在 `Listening` 和音高读数界面之间来回切换。
- 没有稳定音高时，仍保留调音器主体 UI，只显示：
  - `--`
  - `-- Hz`
  - `0 cents` 或空状态指针居中
  - 辅助提示：`等待输入` / `请弹奏单音`
- 有稳定音高时，在同一套 UI 内更新：
  - 音名
  - 频率
  - cents 偏差
  - 指针位置
- 指针、频率和音名更新需要平滑，避免剧烈闪烁。
- 可以保留最近一次稳定识别结果一小段时间，例如 500ms~1000ms，再回到空状态。
- 权限未授予或麦克风不可用时，可以显示单独的错误/权限状态，但普通无声不应切换成完全不同的布局。

## 7. 开发要求

- 在正式改代码前，先基于本文档确认下一轮优化范围。
- 保持现有能力稳定：
  - 节拍器 Start / Stop。
  - Rest 静音。
  - 倒计时结束自动停止。
  - WebView 页面保活。
  - 调音器真实麦克风输入。
- UI 改动后需要运行：

```powershell
D:\flutter\bin\flutter.bat analyze
D:\flutter\bin\flutter.bat test
D:\flutter\bin\flutter.bat build apk --debug
```
