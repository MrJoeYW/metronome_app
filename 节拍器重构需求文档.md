# Flutter 节拍器第三轮优化需求文档

请基于当前 Flutter 节拍器项目进行第三轮优化。本轮先聚焦首页空间、抽屉交互、定时器体验，以及调音器真实功能开发。

## 1. 首页 Stats 区域调整

当前首页底部的 `StatsPanel / PracticeStatsPanel` 不再需要展示。

优化要求：

- 删除首页的 Stats 栏。
- 删除后空出来的空间，不要让 UI 继续挤在一起。
- 首页整体布局从上到下可以更松散一些，让顶部功能栏、轻重拍编辑条、BPM 圆盘、Start 按钮之间的间距更舒展。
- Start 按钮仍然必须完整显示，不能被底部导航遮挡。

后续规划备注：

- 练习时长统计以后会放到其他页面。
- 未来希望做成类似 GitHub contribution calendar 的练琴日历：
  - 每一天一个格子。
  - 当天有练琴记录时格子上色。
  - 练琴时长越长，颜色越深。
  - 当前第三轮可以先不实现该日历，但不要再把练习统计塞在首页。

## 2. 抽屉通用交互优化

当前问题：

- 抽屉默认只打开到屏幕约五分之三位置。
- 如果抽屉内容过长，用户必须先向上拉动抽屉，才能看到底部的 Apply / Confirm / 保存按钮。
- 这会让每次保存设置多一步操作，体验不合理。

优化方向：

- 如果无法稳定控制抽屉内部元素长度，就把 Apply / Confirm / 保存按钮移到抽屉右上角。
- 右上角按钮替换当前关闭按钮位置。
- 关闭能力可以保留为其他轻量入口，例如左上角返回/关闭图标、点击遮罩关闭、下滑关闭等。
- 关键原则：用户打开抽屉后，不需要额外拖动抽屉，就能直接完成应用/保存操作。

适用抽屉：

- Meter 拍号抽屉
- Tone 音色抽屉
- Timer 定时抽屉
- 后续其他设置抽屉

## 3. Meter 拍号抽屉优化

当前问题：

- `Quick meters` 一行明明可以放下 5 个快捷拍型，却实际只排了 4 个一行。
- 这种布局没有必要，视觉上也显得浪费空间。

优化要求：

- `Quick meters` 快捷拍型尽量一行展示 5 个：
  - `2/4`
  - `3/4`
  - `4/4`
  - `5/4`
  - `6/8`
- 在常见手机宽度下优先保证这 5 个可以一行排下。
- 如果极窄屏幕确实放不下，再自然换行。
- 优化当前双滚轮样式，让它更美观、更像正式产品中的选择器。
- 滚轮应清楚表达：
  - 左侧：拍数 `1~16`
  - 右侧：时值 `1/2/4/8/16/32`

## 4. Timer 定时抽屉优化

当前问题：

- `Duration` 预设项布局和 Meter 的快捷项类似，也存在没有充分利用横向空间的问题。
- 当前使用加减号选择自定义时间，如果用户想选择较大的分钟数，需要点很多次，不合理。

优化要求：

- Timer 抽屉不要再用 `+ / -` 选择自定义分钟数。
- 改为滚轮选择分钟数。
- 最大支持 `999` 分钟。
- 保留常用预设：
  - `1`
  - `3`
  - `5`
  - `10`
  - `15`
- 常用预设尽量一行展示。
- 优化滚轮视觉样式，使其比当前 Meter 抽屉里的滚轮更好看。
- `Off / Countdown` segmented control 仍需保持固定尺寸，选中状态不能导致布局尺寸变化。
- Apply 后仍然要真正启动倒计时。
- 首页定时按钮显示剩余时间。
- 倒计时结束后停止节拍器。

## 5. 调音器真实功能开发

当前调音器只显示“调音器开发中”，第三轮需要开始开发真实调音器功能。

目标：

- 接入麦克风输入。
- 实时识别当前音高。
- 展示：
  - 音名，例如 `A4`
  - 频率，例如 `440Hz`
  - 偏差，例如 `+3 cents`
- 提供直观的调音指示 UI，例如指针、刻度条或居中偏差条。

基础要求：

- 未获得麦克风权限时，要显示权限提示。
- 麦克风不可用时，要有明确提示。
- 不要展示假数据。
- 如果暂时无法完整实现高精度识别，也要至少完成真实音频输入、基础频率检测和 UI 状态流转。

可后续扩展但本轮可预留：

- 标准音 A4 可调。
- 吉他/贝斯模式。
- 历史音高曲线。
- 自动识别弦名。

## 6. 开发要求

- 在正式改代码前，先基于本文档确认第三轮优化范围。
- 优先保证现有节拍器播放逻辑稳定，不要破坏 Start/Stop、Rest 静音、倒计时停止、WebView 悬浮按钮等第二轮已完成能力。
- UI 改动后需要重新运行：

```powershell
D:\flutter\bin\flutter.bat analyze
D:\flutter\bin\flutter.bat test
D:\flutter\bin\flutter.bat build apk --debug
```

- 如果新增调音器依赖、麦克风权限、Android/iOS 平台配置，必须同步更新：
  - `pubspec.yaml`
  - Android Manifest / iOS plist 等平台权限
  - `PROJECT_STRUCTURE.md`

## 7. 文档维护要求

- 如果本轮确认后进入开发，完成代码修改时必须同步更新 `PROJECT_STRUCTURE.md`。
- `PROJECT_STRUCTURE.md` 需要记录新增调音器能力、权限、依赖、主要文件职责和验证命令。
