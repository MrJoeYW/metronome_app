# Flutter 节拍器下一轮重构提示词

请基于当前 Flutter 节拍器项目继续优化。本轮只做两个方向：

1. 将底部“设置”页改造成“我的”页，并完善练习记录展示。
2. 完善节拍器主页的轻重拍编辑条，长按单拍时弹出音符类型选择抽屉。

已完成的 WebView 自定义网址、WebView 返回、半透明悬浮 Start/Stop、调音器 UI 稳定性、保存/加载配置迁移、主页 Save/Load 图标入口、主页布局和代码拆分，不再重复列入本轮范围。修改后仍需保持这些能力稳定可用。

## 1. 底部“设置”页改为“我的”

当前底部第三个 Tab 是“设置”，页面主要展示练习统计。新的产品方向是把它改成更个人化的“我的”页。

优化要求：

- 底部第三个 Tab 文案改为“我的”。
- 图标可使用 `Icons.person_rounded`、`Icons.account_circle_rounded` 或符合现有风格的个人中心图标。
- 页面标题从 `Practice Stats` 调整为“我的”或“练习记录”，最终命名可根据现有 UI 风格决定。
- 页面仍保留当前已有的练习统计信息，例如：
  - 今日练习时长
  - 当前练习会话时长
  - 近期记录数量
  - 累计练习时长
  - 平均 BPM
- 页面新增类似 GitHub contribution calendar 的“练琴日历”。

## 2. 练琴日历

目标：用类似 GitHub contribution calendar 的方式展示每日练琴情况，让用户快速看到练习连续性和强度。

功能要求：

- 在“我的”页新增一个练琴日历区域。
- 日历以小方块网格展示每日练习强度，视觉参考 GitHub contribution calendar。
- 每个方块代表一天。
- 方块颜色深浅根据当天练习总时长变化：
  - 无练习：低对比度空状态
  - 少量练习：浅色
  - 中等练习：中等强调色
  - 较长练习：高强调色
- 建议默认展示最近 12 周或最近 3 个月，具体范围根据手机屏幕宽度自适应。
- 日历需要适配移动端：
  - 小方块尺寸稳定，不因文字变化导致跳动。
  - 横向空间不足时可以减少展示周数，或允许横向滚动。
  - 不要让日历挤压现有统计卡片。
- 日历区域应有简洁标题，例如“练琴日历”。
- 可提供简单图例，例如 `Less` 到 `More`，但不要占用过多空间。

交互要求：

- 点击某一天的日历方块，弹出 `AlertDialog` 展示当天练习详情。
- 弹窗内容至少包含：
  - 日期
  - 当天总练习时长
  - 当天练习次数
  - 当天练习记录列表
- 练习记录列表可展示：
  - 开始时间或记录时间
  - 单次练习时长
  - 平均 BPM
- 如果当天没有练习记录，弹窗显示空状态提示，例如“这一天还没有练习记录”。
- 弹窗需要有明确关闭按钮。

数据要求：

- 复用当前 `PracticeLogs` 表。
- 如果当前数据库封装只支持今日汇总和最近记录，需要补充查询能力：
  - 按日期范围汇总每日练习时长。
  - 按某一天查询练习明细。
- 不要破坏现有练习记录写入逻辑。
- 当前播放中的练习会话如果尚未落库，页面展示时仍应尽量把正在进行的会话计入今日显示，避免用户看到明显滞后的今日时长。
- 如果要新增模型，例如 `PracticeDaySummary`，请放在合适的模型或数据库封装位置，并在 `PROJECT_STRUCTURE.md` 记录。

## 3. 长按轻重拍编辑条选择音符类型

当前主页轻重拍编辑条支持点击单拍循环轻重类型，长按只打开预留编辑面板。本轮要把长按行为做成真正可用的“音符类型选择”抽屉。

目标：

- 用户长按某一拍时，打开底部抽屉。
- 抽屉中展示多个音符类型选项。
- 用户点击某个音符类型后，将该类型应用到当前拍。
- 选择后关闭抽屉或给出明确确认方式，具体交互可根据现有抽屉风格决定。

音符类型范围：

- 至少包含常见音符：
  - 全音符
  - 二分音符
  - 四分音符
  - 八分音符
  - 十六分音符
  - 三十二分音符
- 可以包含常见附点和三连音组合：
  - 附点二分音符
  - 附点四分音符
  - 附点八分音符
  - 八分三连音
  - 十六分三连音
  - 前八后十六
  - 前十六后八
  - 其他常用节奏型

视觉要求：

- 音符类型选择界面参考用户提供的图片：深色背景、网格布局、每个格子里居中展示一个音符或节奏型符号。
- 建议使用 4 列网格或根据屏幕宽度自适应。
- 每个选项应有稳定尺寸，避免图标或文字导致布局跳动。
- 当前已选类型需要有选中态，例如边框、背景色或强调色。
- 音符符号优先使用清晰稳定的视觉表达：
  - 如果项目已有音乐符号资源，优先复用。
  - 如果没有资源，可先用 Unicode 音乐符号、简洁文本标签或自绘 CustomPainter 实现。
  - 不要为了展示音符类型引入过重依赖。
- 参考图中的网格只作为风格参考，不要求完全复刻每一个复杂符号，但要让用户能清楚区分不同节奏型。

交互要求：

- 保留当前“点击单拍循环轻重拍类型”的能力。
- 长按单拍只负责打开音符类型选择，不应改变当前轻重拍类型。
- 抽屉标题需要说明正在编辑第几拍，例如 `Beat 1` 或“第 1 拍”。
- 选择音符类型后，主页轻重拍编辑条需要能反映该拍的音符类型。
- 如果当前 Android 原生节拍引擎尚未支持每拍不同音符类型，则先完成 Flutter UI 状态、持久化模型和配置下发字段预留，并在 `PROJECT_STRUCTURE.md` 说明原生层待接入。

数据和配置要求：

- 当前 `MetronomeConfig` 已包含 `subdivisionType` 和 `beatTypes`。
- 本轮需要评估是否新增每拍音符类型字段，例如：
  - `beatNoteTypes`
  - 或更明确的 `beatRhythmTypes`
- 如果新增字段，需要：
  - 支持 UI 状态维护。
  - 支持保存/加载配置。
  - 支持最近一次设置恢复。
  - 支持下发给 Android MethodChannel，即使原生层暂时只缓存或忽略也要保持兼容。
  - 支持数据库迁移，不破坏旧用户数据。
- 如果先不接入 Android 播放逻辑，必须保证现有节拍器播放、Rest 静音、细分、TTS、振动不回退。

## 4. 开发要求

- 修改前先阅读当前代码结构，优先沿用现有 `pages/`、`widgets/`、`sheets/`、`models/` 的拆分方式。
- UI 风格要和当前 App 保持一致：深色背景、8px 左右圆角、清晰的图标按钮和紧凑控件。
- 不要把“我的”页做成营销页或说明页，首屏直接展示用户练习数据和练琴日历。
- 保持现有能力稳定：
  - 节拍器 Start / Stop
  - Rest 静音
  - 倒计时结束自动停止
  - WebView 页面保活和自定义网址
  - WebView 返回按钮
  - WebView 半透明悬浮 Start / Stop
  - 调音器真实麦克风输入和稳定 UI
  - 已保存配置的保存、加载、删除和恢复
  - 今日练习统计展示

```powershell
D:\flutter\bin\flutter.bat analyze

D:\flutter\bin\flutter.bat build apk --debug
```
